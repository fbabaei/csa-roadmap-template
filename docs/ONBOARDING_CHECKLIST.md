# Onboarding Checklist

## Before week 1

- [ ] Define owner and reviewers
- [ ] Choose pace (10/6/14-16 weeks)
- [ ] Select cloud environment/subscription(s)
- [ ] Define project slots A to D
- [ ] Confirm documentation location and naming conventions

## Setup

- [ ] Clone or copy template folder
- [ ] Fill `docs/PLAN_TEMPLATE.md`
- [ ] Fill `projects/INDEX.md`
- [ ] Fill `docs/EXECUTIVE_SUMMARY_TEMPLATE.md`
- [ ] Fill `docs/RESOURCE_MATRIX_TEMPLATE.md`

## Weekly execution

- [ ] Update `week-XX/WEEK_ASSIGNMENT.md`
- [ ] Update `week-XX/STATUS.md`
- [ ] Add evidence links in week folder
- [ ] Mark status (`Not Started`, `In Progress`, `Completed`, `Blocked`)

## Monthly/phase review

- [ ] Run architecture review
- [ ] Update risk register
- [ ] Share executive summary
- [ ] Record lessons learned
