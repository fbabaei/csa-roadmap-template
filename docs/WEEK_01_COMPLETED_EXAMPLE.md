# Week-01 Completed Example

This file provides an internal example of what a completed week-01 can look like.

## Purpose

Use this as a quality bar reference when filling:

- week-01/WEEK_ASSIGNMENT.md
- week-01/STATUS.md
- docs/EXECUTIVE_SUMMARY_TEMPLATE.md
- docs/RESOURCE_MATRIX_TEMPLATE.md

## Example inputs

- Owner: CSA Team Alpha
- Program pace: Standard (10 weeks)
- Project A: Platform landing zone and IaC baseline

## Example week-01 assignment quality

- Objective: Establish architecture foundation decisions and initial IaC baseline.
- Deliverables:
  - Initial architecture context diagram
  - v1 network and identity baseline in IaC
  - ADR draft for naming and environment strategy
- Validation criteria:
  - Diagram reviewed by lead architect
  - IaC deploy plan executes successfully in non-prod
  - ADR reviewed by team
- Evidence links:
  - Link to diagram source
  - Link to IaC module path
  - Link to ADR markdown

## Example week-01 status quality

- Status: Completed
- Highlights:
  - Defined target-state architecture boundaries
  - Implemented baseline IaC modules for network and identity
  - Aligned naming conventions with platform team
- Blockers:
  - Access delay to shared subscription for one day
- Risks:
  - Pending policy exceptions for one resource type
- Next actions:
  - Start week-02 security and policy controls
  - Add evidence for policy remediation walkthrough

## Example executive update quality

- Current status: On track
- This period highlights:
  - Foundation architecture approved
  - IaC baseline deployed in non-prod
- Upcoming milestones:
  - Week-02 policy and governance controls
- Support needed:
  - Security review slot on Monday

## Example resource matrix quality

For each critical resource type, mark one of:

- Yes: actively used in this slot/week
- Planned: expected in next phase
- No: not needed for this slot

Use brief notes for non-obvious choices to improve handover quality.
