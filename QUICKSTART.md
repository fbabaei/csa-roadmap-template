# Quick Start - Day One Execution

This guide helps a new user start this template from zero and complete day one setup with a usable roadmap structure.

## Day one goal

By the end of day one, you should have:

- a filled plan baseline
- named project slots A-D
- week-01 assignment and status updated
- an executive summary draft
- a first resource matrix draft

## Prerequisites

- Access to this repository
- A text editor (VS Code recommended)
- 60 to 90 minutes of focused time

## Step-by-step

## Step 1 - Open the template portal (5 min)

- Run `python scripts/start_portal.py` from the repository root.
- Or open `index.html` in this folder.
- Review Executive Mode and audience paths.
- Open these three files from the portal:
  - `docs/PLAN_TEMPLATE.md`
  - `projects/INDEX.md`
  - `docs/ONBOARDING_CHECKLIST.md`

## Step 2 - Define your program metadata (10 min)

In `docs/PLAN_TEMPLATE.md`, fill:

- Owner
- Team or cohort
- Start date
- Target end date
- Pace (10, 6, or 14-16 weeks)

Then write a one-paragraph goal statement.

## Step 3 - Map your real projects to slots A-D (15 min)

Open `projects/INDEX.md` and assign:

- Project A: architecture and IaC
- Project B: AI application pattern
- Project C: operations/tooling
- Project D: delivery reference implementation

Then open each slot README under `projects/` and add specific outcomes.

## Step 4 - Set week-01 execution baseline (15 min)

Open:

- `week-01/WEEK_ASSIGNMENT.md`
- `week-01/STATUS.md`

Fill objective, deliverables, and checklist.
Set status to `In Progress` after you start work.

## Step 5 - Build executive visibility (10 min)

Open `docs/EXECUTIVE_SUMMARY_TEMPLATE.md` and fill:

- current status
- this period highlights
- upcoming milestones
- support needed

## Step 6 - Start your resource mapping (10 min)

Open `docs/RESOURCE_MATRIX_TEMPLATE.md`.
Mark the first draft of which resources are used by each project slot.
Use symbols like `Yes`, `No`, or `Planned`.

## Step 7 - Finish day one closeout (5 min)

Confirm all items in `docs/ONBOARDING_CHECKLIST.md` under Setup are checked.

## Day one done criteria

You are done for day one when:

- `docs/PLAN_TEMPLATE.md` has owner, timeline, and week table started
- `projects/INDEX.md` has your actual project names
- `week-01/STATUS.md` is updated and not empty
- `docs/EXECUTIVE_SUMMARY_TEMPLATE.md` has a first draft
- `docs/RESOURCE_MATRIX_TEMPLATE.md` has initial mapping

## Day two recommendation

- Complete week-01 deliverables
- Collect first evidence links
- Update week-01 status and blockers
- Prepare week-02 assignment before starting implementation
